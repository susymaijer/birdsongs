Program Bugs of the CD-ROM "Birds of Venezuela"

by Sjoerd Mayer

  • Read this if the installation fails
  • This bug only comes into play if you try to copy the entire CD-ROM to your hard disk (see "Copying the complete CD-ROM to your hard disk" in the Readme file - which is opened when you click "Read Me" in the Help menu).
    The problem is that the program does not understand Windows 95-style long file names, so that if you copy the CD-ROM contents to a directory with a long file name, it will not be able to find the sounds and photographs.
    There are two solutions:
    1. Copy to a directory with an old-fashioned name (ie 8 characters or less, no spaces, no "funny characters"). The default directory ("c:\venbirds") to which the CD-ROM is installed would be a good choice.
    2. Or if you copy to a directory with a long name, tell the program its "short name equivalent" when it prompts you for the CD-ROM drive (example: "c:\mydocu~1" is the short version of "c:\My Documents").
